## Spacing Scale

Declared values (multiples of 4 only — matches existing Tailwind v4 default scale used across `components/landing/*` and `components/directory/*`):

|-------|----------|-------|---------------------|

| sm | `gap-2` / `p-2` | 8px | Inline element spacing (icon + text on CTA buttons, step counter gaps) |

## Typography

Pre-populated from existing codebase patterns (`components/landing/Hero.tsx`, `components/landing/HowItWorks.tsx`, `components/directory/DirectoryEmptyState.tsx`). **4 sizes, 2 weights** — meets checker's "3–4 sizes, 2 weights max" rule.

| Role | Size | Tailwind | Weight | Line Height | Font Family |
|------|------|----------|--------|-------------|-------------|
| Display (wizard step headline) | 28px (`text-2xl` sm:`text-3xl`) | `text-2xl sm:text-3xl` | 700 (bold) | 1.2 (`leading-tight`) | Lora serif (`font-serif`) |
| Heading (section/sub labels, progress "Step 1 of 3") | 20px default (`text-xl`); 18px sub-variant (`text-lg`) permitted for Step 1 inner section labels inside the checklist region — same role, same weight, same family, just a smaller render in tight layouts | `text-xl` (default), `text-lg` (sub-variant) | 700 (bold) | 1.3 (`leading-snug`) | Lora serif (`font-serif`) |
| Body (step explanation paragraph) | 16px (`text-base`) | `text-base` | 400 (regular) | 1.5 (`leading-relaxed`) | Inter sans (`font-sans`) |
| Label / micro (progress eyebrow "How it works", "Step 1 of 3" counter) | 12px (`text-xs`) | `text-xs uppercase tracking-[0.18em]` | 700 (bold) | 1.4 | Inter sans (`font-sans`) |

Rules (locked by precedent):
- Wizard step headlines MUST be `font-serif` + `font-bold` — all existing H1/H2 use Lora bold (`Hero` L46, `HowItWorks` L43, `DirectoryEmptyState` L11).
- Body paragraphs MUST be `text-base` + `leading-relaxed` (or explicit `leading-[1.5]`) — matches `DirectoryEmptyState` L14 and `HowItWorks` L46.
- Weights used: **400 (regular)** for body + **700 (bold)** for headings, progress eyebrows, and CTA button labels. Exactly 2 weights. CTA labels use `font-bold` (matches `Hero.tsx` bold CTA precedent) — no third weight is introduced.
- The `text-lg` (18px) size listed under Heading is NOT a separate size tier — it is a tight-layout sub-variant of the 20px Heading role, using the same font family and weight. Total distinct size tiers = 4 (28px display, 20px/18px heading, 16px body, 12px micro).

## Color

60/30/10 split matches the existing sage/forest/clay system. All values already present in `app/globals.css` `@theme`.

| Role | Token / Value | Usage in this phase |
|------|---------------|---------------------|
| Dominant (60%) — page surface | `--color-sage-bg` = `#eef3e8` | `/onboarding` route body background (inherits from root `layout.tsx` `bg-sage-bg`) |
| Secondary (30%) — card surface | `--color-sage-pale` = `#f4f7f0` with 1px `border-sage-light` (`#dfe8d5`) ring | Wizard `Card` container (matches `DirectoryEmptyState` L9 `bg-sage-pale border-sage-light`) |
| Accent (10%) — primary CTA + active-state focus | `--color-clay` = `#c4956a` (shadcn `--color-primary` is aliased to clay) | Reserved STRICTLY for: (1) primary "Next" / "Edit my profile →" / "Browse the directory →" / "Find someone to contact →" buttons, (2) active progress-indicator dot/pill, (3) AppNav "Finish setup →" badge accent when rendered as pill, (4) focus ring on interactive elements. |
| Forest deep — text | `--color-forest-deep` = `#1e4420` | Step headlines, body paragraph color (inherits from root `layout.tsx text-forest-deep`) |
| Forest mid — subtle text + step-complete checks | `--color-forest-mid` = `#3a7032` | Progress counter eyebrow, inactive progress dots, "complete" check icon color (matches `ProfileCompletenessChecklist` L22 `text-forest-mid`) |
| Destructive | shadcn `--destructive` (oklch(0.577 0.245 27.325)) | Not used this phase (no destructive actions — dismiss is non-destructive). |

Accent reserved for:
1. Primary CTA button background (`bg-clay hover:bg-clay/90 text-sage-bg`) — exactly 1 primary CTA per step
2. Active progress-indicator dot (the current step only)
3. Focus rings on `<Link>`/`<Button>` inside the wizard card (via `--color-ring: var(--color-clay)`)
4. Optional AppNav "Finish setup" subtle dot badge (6–8px solid clay dot, not a full-pill fill)

Accent MUST NOT appear on: body text, checklist rows, card borders, step counter eyebrow, "Skip for now" link (which must use `text-forest-mid` underline).

## Copywriting Contract

Voice: warm, community, plain-spoken, Georgia-specific, consistent with legacy `index.html` and `Hero.tsx` / `HowItWorks.tsx` tone ("No money. No middlemen." / "Georgians helping Georgians"). Avoid corporate onboarding clichés ("Let's get you set up!"). Prefer contractions and direct second-person.

### Wizard-level

| Element | Copy |
|---------|------|
| Progress eyebrow | `Welcome to Barterkin` (on all three steps, above step headline) |
| Progress counter | `Step {N} of 3` (where N is 1, 2, 3) |
| Dismiss link (footer of wizard card, on all steps) | `Skip for now` (underlined, text-forest-mid, leads to `/directory`, does NOT mark wizard complete per D-16) |

### Step 1 — Profile

| Element | Copy |
|---------|------|
| Headline | `First, finish your profile.` |
| Body | `Neighbors find you by county, category, and the skills you offer. Fill in these five and you're on the map.` |
| Embedded | `<ProfileCompletenessChecklist />` rendered live (reuse existing component per D-06) |
| Primary CTA (when incomplete) | `Edit my profile →` (navigates to `/profile/edit?returnTo=/onboarding?step=1`) |
| Next button state (incomplete) | Disabled. Label: `Next` (greyed, with hover tooltip `Finish all five to continue` using existing `Tooltip` primitive) |
| Next button state (complete — all 5 green) | Enabled. Label: `Next: browse the directory →` |

### Step 2 — Directory

| Element | Copy |
|---------|------|
| Headline | `Next, browse your neighbors.` |
| Body | `Filter by county and category. Search by keyword. Every profile you see is a real Georgian with an email-verified account.` |
| Primary CTA | `Browse the directory →` (opens `/directory` in same tab per D-07) |
| Next button (always active per D-15) | `Next: send your first hello →` |

### Step 3 — Contact

| Element | Copy |
|---------|------|
| Headline | `Finally, say hello.` |
| Body | `Find someone whose skills match yours. Send a short note through Barterkin — their reply lands in your inbox. No fees. No middlemen.` |
| Primary CTA | `Find someone to contact →` (opens `/directory` per D-08) |
| Finish button | `I'm all set` (marks `onboarding_completed_at = now()` via Server Action on step render per D-11; redirects to `/directory`) |

### AppNav affordance (persists until `onboarding_completed_at IS NOT NULL`)

| Element | Copy |
|---------|------|
| Resume link label | `Finish setup →` (plain text link, `text-forest-mid`; optional 6px solid `bg-clay` dot adjacent as subtle badge) |
| Resume link target | `/onboarding?step={N}` where N is inferred from profile completeness server-side (complete = last-reached step; incomplete = 1) |

### Empty / Edge States

| Element | Copy |
|---------|------|
| Empty state heading | N/A — wizard always has content; no empty state exists (steps are instructional, not data-driven) |
| Empty state body | N/A |
| Error state heading | `We couldn't load your progress.` |
| Error state body | `Refresh the page or skip ahead — we'll remember where you left off.` (shown inside `Alert` variant="destructive" above step content if profile fetch throws) |
| Destructive confirmation | N/A — there are no destructive actions in this phase. Dismissing via "Skip for now" is non-destructive (user can resume). |

## Interaction Contract

### Layout (desktop ≥768px)
- Centered `Card` at `max-w-xl` (576px), `mx-auto`, `mt-12 sm:mt-16`
- Card interior: `p-8 md:p-12`, `space-y-6`
- Progress indicator: top of card, `mb-6`
- Step body: `space-y-4`
- CTA row: `mt-8 flex flex-col gap-3 sm:flex-row sm:justify-end`
- Footer separator + "Skip for now": `mt-8 pt-6 border-t border-sage-light text-center`

### Layout (mobile ≥360px per LAND-03 precedent)
- Card: `mx-4` (16px gutter), full-width up to `max-w-xl`
- CTAs stack vertically (`flex-col`), full-width, `h-11` tap target
- Progress indicator dots remain horizontal (compact)

### States

| State | Visual |
|-------|--------|
| Step 1 loading | `Skeleton` rows where checklist renders; CTAs visible but Next disabled |
| Step 1 incomplete (0–4 green) | Next button disabled (`disabled` + `opacity-50` + `cursor-not-allowed`); Tooltip on hover |
| Step 1 complete (5/5 green) | Next button enabled; subtle `ring-1 ring-clay/30` pulse on Next for 2s (CSS transition only, not continuous) |
| Step 2/3 | Next / Finish always enabled |
| Error | `Alert variant="destructive"` above content; step body still renders in degraded state |
| Mid-step navigation via progress dots | Dots are clickable only for already-visited/completed steps (forward navigation still requires Next) |

### Focus order
1. Headline (programmatic focus on step change for screen-reader announcement via `tabIndex=-1` + `.focus()`)
2. Primary CTA (`Edit my profile →` / `Browse the directory →` / `Find someone to contact →`)
3. Next / Finish button
4. "Skip for now" link

### Keyboard
- `Enter` on focused primary CTA activates it
- `Escape` on the wizard triggers "Skip for now" redirect to `/directory` (with same non-completion semantics)
- Progress-indicator dots reachable via `Tab` only when they are clickable (visited steps)

## Accessibility Contract

- All interactive elements have visible focus ring (clay, 2px) via shadcn `--ring` token
- Primary CTAs have `aria-describedby` pointing to step body when copy alone is ambiguous
- Progress indicator: `role="progressbar"` + `aria-valuenow={currentStep}` + `aria-valuemin={1}` + `aria-valuemax={3}` + `aria-label="Onboarding progress"`
- Disabled Next button uses `aria-disabled="true"` + real `disabled` attribute; tooltip is `role="tooltip"` announced via `aria-describedby`
- Step transitions announce via `aria-live="polite"` region containing `Step {N} of 3: {headline}`
- Color contrast: forest-deep (`#1e4420`) on sage-pale (`#f4f7f0`) = 10.8:1 (AAA); clay (`#c4956a`) on sage-bg (`#eef3e8`) = 2.8:1 — **clay is background-only on large text (CTA labels use sage-bg foreground on clay background = 4.9:1, AA for large ≥18pt bold)**. No body text ever on clay.
- Mobile tap targets: 44×44px minimum via `h-11` — matches LAND-03 precedent
