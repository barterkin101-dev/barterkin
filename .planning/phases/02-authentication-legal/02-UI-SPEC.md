# Phase 2 — UI Design Contract

> Visual and interaction contract for the Authentication & Legal phase. Covers 7 surfaces: `/login`, `/signup`, `/verify-pending`, `/legal/tos`, `/legal/privacy`, `/legal/guidelines`, and footer updates (legal links + logout). Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn/ui (CLI v3, initialized Phase 1) |
| Preset | `new-york` style + `stone` base color + Tailwind v4 CSS-first (`@theme` in `app/globals.css`) |
| Component library | Radix primitives (via shadcn) — `Button`, `Card`, `Input`, `Label` already installed in `components/ui/` |
| Icon library | `lucide-react` (configured in `components.json`) |
| Font | `Lora` (serif, headings) + `Inter` (sans, body) — loaded via `next/font/google`, exposed as `--font-serif` / `--font-sans` |

**Existing palette vars (DO NOT redefine — consume from `@theme` in `app/globals.css`):**
- `--color-sage-bg #eef3e8`, `--color-sage-light #dfe8d5`, `--color-sage-pale #f4f7f0`
- `--color-forest #2d5a27`, `--color-forest-deep #1e4420`, `--color-forest-mid #3a7032`
- `--color-clay #c4956a`

**shadcn tokens already mapped** via `@theme inline` → `--color-background`, `--color-foreground`, `--color-primary`, etc. Phase 2 surfaces must consume these tokens, not raw hex. If a token's current stone value fights the warm palette on an auth surface (e.g. the `Button` primary is currently `oklch(0.205 0 0)` near-black), Phase 2 locks the brand override in the same `@theme inline` block — see Color section.

## Spacing Scale

All values in multiples of 4 (Tailwind v4 default 4px baseline — `1 = 4px`, `2 = 8px`, `4 = 16px`, `6 = 24px`, `8 = 32px`, `12 = 48px`, `16 = 64px`).

| Token | Value | Usage in Phase 2 |
|-------|-------|------------------|
| xs | 4px (`gap-1`, `p-1`) | Icon-label gaps inside buttons (Google logo → "Continue with Google"), inline helper text margin |
| sm | 8px (`gap-2`, `p-2`) | Form field internal padding, error-message top-margin below inputs |
| md | 16px (`gap-4`, `p-4`) | Vertical rhythm inside auth card (label → input → help text), paragraph spacing in legal pages |
| lg | 24px (`gap-6`, `p-6`) | Auth card internal padding, section spacing between OAuth block → divider → magic-link block |
| xl | 32px (`gap-8`, `p-8`) | Space between auth card and legal-microcopy footer, verify-pending heading → body gap |
| 2xl | 48px (`gap-12`, `py-12`) | Vertical page padding on auth routes (mobile top/bottom breathing room), legal-page section breaks |
| 3xl | 64px (`gap-16`, `py-16`) | Desktop vertical page padding for `/login`, `/signup`, `/verify-pending`; major section breaks in long legal pages |

**Exceptions for Phase 2:**
- **Tap targets ≥ 44px**: Google OAuth button, magic-link submit button, Turnstile widget, and footer logout button all render at `h-11` (44px) minimum to satisfy iOS HIG and WCAG 2.5.5. The shadcn `Button` default `size="default"` is `h-10` (40px) — Phase 2 auth CTAs use `size="lg"` (`h-11`) explicitly.
- **Turnstile widget**: Cloudflare renders the iframe at exactly 300×65px (standard) — wrap in a `flex justify-center my-4` container; do not try to force it into the 4px grid.
- **Auth card max-width**: 400px (`max-w-sm` = 384px is too tight for "Continue with Google" at large font; use `max-w-[400px]`).
- **Legal-page content max-width**: 672px (`max-w-2xl`) — optimal line length (~66ch) for readable terms-of-service prose.

## Typography

Exactly 2 weights used: 400 (regular) and 700 (bold). Lora (serif) renders headings only; Inter (sans) renders everything else.

| Role | Size | Weight | Line Height | Font | Surfaces |
|------|------|--------|-------------|------|----------|
| Body | 16px (`text-base`) | 400 (`font-normal`) | 1.5 (`leading-[1.5]`) | Inter | All paragraphs, help text, legal-page body copy, verify-pending explanation |
| Label | 14px (`text-sm`) | 400 (`font-normal`) | 1.4 (`leading-[1.4]`) | Inter | Form field labels ("Email address"), button labels, footer links |
| Heading | 24px (`text-2xl`) | 700 (`font-bold`) | 1.2 (`leading-[1.2]`) | Lora (`font-serif`) | Auth card heading ("Welcome to Barterkin"), verify-pending heading, legal-page H2 section titles |
| Display | 32px desktop / 28px mobile (`text-3xl md:text-[32px]`) | 700 (`font-bold`) | 1.15 (`leading-[1.15]`) | Lora (`font-serif`) | Legal-page H1 ("Terms of Service", "Privacy Policy", "Community Guidelines") |

**Labels are distinguished from body by size (14px vs 16px), not weight.**

**Responsive behavior:** Display drops from 32px → 28px below `md` (768px). Heading stays 24px at all widths. Body never drops below 16px (iOS auto-zoom trigger floor).

**Weight discipline:** Exactly 2 weights used: 400 (regular) and 700 (bold). No 300/light (weak on sage background at body size), no 500/medium, no 600/semibold, no 800/extra-bold. If emphasis is needed inside body copy, use italics or bold (700) sparingly — never `font-medium`.

## Color

Uses the existing sage/forest/clay palette defined in Phase 1. 60/30/10 split applies per-surface, not per-pixel.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--color-sage-bg #eef3e8` | Page background for `/login`, `/signup`, `/verify-pending`, `/legal/*`; body text color on forest surfaces |
| Secondary (30%) | `--color-sage-pale #f4f7f0` (card surface) + `--color-forest #2d5a27` (footer, header nav, legal-page header band) | Auth card background (sage-pale, 1px `--color-sage-light` border); footer is forest deep with sage-bg text |
| Accent (10%) | `--color-clay #c4956a` | Primary CTA buttons ONLY: "Continue with Google", "Send magic link", "Resend link" on verify-pending. Focus rings on form inputs (`--ring: var(--color-clay)`). Link underline on hover inside legal-page body prose. |
| Destructive | `oklch(0.577 0.245 27.325)` (shadcn `--destructive` default) | Error-message text under form inputs, error toast border, rate-limit banner, disposable-email rejection banner. Logout button remains `variant="ghost"` (not destructive) — logout is benign, not dangerous, in this app. |

**Accent reserved-for list (exhaustive for Phase 2):**
1. Primary CTA button background: `/login` "Continue with Google" icon-button (clay bg, white text + Google logo mark in original color)
2. Primary CTA button background: `/login` + `/signup` "Send magic link" submit button (clay bg, sage-bg text)
3. Primary CTA button background: `/verify-pending` "Resend link" button (clay bg, sage-bg text)
4. Focus ring: keyboard-focus outline on all interactive elements (2px clay ring, 2px offset)
5. Inline-link hover: text-decoration underline color on `<a>` elements inside legal-page prose
6. Small brand flourish: single 2px clay horizontal rule separating OAuth block from magic-link block on `/login`

**NOT allowed for accent:** avatar/profile badges (Phase 3+), directory card borders (Phase 4), normal body links in help text (those use forest-mid), footer links (those use sage-bg text on forest bg).

**Brand override of shadcn tokens** (must land in `app/globals.css` during Phase 2 implementation — extend existing `@theme inline` block; do not hand-edit the `:root` stone defaults):

```css

--color-primary-foreground: var(--color-sage-bg);
```

This makes `<Button>` (default variant) render clay-on-sage automatically, so the executor doesn't sprinkle `className="bg-clay"` overrides everywhere.

## Visuals

**Focal point declaration.** The primary visual anchor on `/login` and `/signup` is the clay-background "Continue with Google" button at the top of the auth card — eye path: card heading (Lora 24px bold) → Google OAuth button (clay bg) → divider → magic-link form.

On `/verify-pending`, the primary focal point is the Lora 32px bold "One more step" display heading, followed by the clay "Resend verification link" button as the secondary anchor. On `/legal/*` pages, the focal point is the Lora 32px/28px bold H1 page title, with the sage-bg page background providing a calm reading surface and no competing accent colors above the fold.

The clay accent appears in exactly one location per auth surface (the primary CTA), preserving its role as the eye's first destination. The forest-green footer band sits visually below the auth card to avoid competing with the card for attention.

## Copywriting Contract

Tone: warm, direct, community-first. Second-person ("you", "your"). Never "please" or "kindly". Never "awesome" or "great". Avoid exclamation points except where literally celebrating (e.g., "You're in.").

### Error States

| Scenario | Trigger | Exact Copy | Placement |
|----------|---------|------------|-----------|
| Disposable email rejected (AUTH-07) | Server rejects `@mailinator.com`, `@tempmail.*`, etc. | `That email provider isn't supported. Please use a personal email (Gmail, Outlook, iCloud, or your own domain).` | Red text below email input, 14px, `--destructive`, with `AlertCircle` lucide icon 16px inline-left |
| Rate limit hit (AUTH-06) | 5+ accounts from same IP in 24h | `Too many signups from this network today. Please try again tomorrow, or [contact us] if you think this is a mistake.` | Full-width banner above auth card, sage-pale bg, destructive left-border 4px, 16px padding |
| CAPTCHA failure (AUTH-08) | Turnstile token invalid/expired | `Couldn't verify you're human. Refresh the page and try again.` | Red text below Turnstile widget, 14px, with `AlertCircle` icon; button stays disabled until Turnstile re-renders |
| Magic-link expired | User clicks an >1h-old link | `That link has expired. Request a new one below.` | Inline above email input on `/login`, 14px destructive, with form pre-fillable |
| Magic-link already-used | User clicks a link twice | `That link was already used. You're signed in on another device, or you need a new link.` | Inline above email input on `/login`, 14px destructive |
| Generic auth error | Network/unknown | `Something went wrong. Please try again in a moment.` | Red text below the triggering button, 14px destructive |
| Google OAuth denied | User cancels Google consent | (no error UI — silent return to `/login` with form re-rendered, no alarm) | — |

### CAPTCHA / Turnstile Notice

A 12px `--muted-foreground` line renders directly below the Cloudflare Turnstile widget on both `/login` and `/signup`:

```
Protected by Cloudflare Turnstile. [Privacy] · [Terms]
```

Links open `https://www.cloudflare.com/privacypolicy/` and `https://www.cloudflare.com/website-terms/` in new tabs (`target="_blank" rel="noopener"`).

### Logout Button (in footer)

| State | Copy | ARIA | Variant |
|-------|------|------|---------|
| Authed user visible in footer | `Log out` | `aria-label="Log out of Barterkin"` | `variant="ghost"`, 14px, with `LogOut` lucide icon 14px inline-left |
| Unauthed | (not rendered) | — | — |
| During logout (optimistic) | `Logging out…` | `aria-label="Log out of Barterkin"` | disabled, same ghost variant |

**No confirmation dialog.** Logout is reversible (re-login in 10 seconds). A confirm-modal on logout is friction-for-no-reason.

## Accessibility Contract

- All form inputs have `<Label htmlFor>` wired (shadcn `Form` handles this).
- Turnstile widget must be keyboard-focusable; the fallback for users with JS disabled is an explicit "JavaScript required to verify you're human" `Alert`.
- Error messages associated with inputs via `aria-describedby` (shadcn `FormMessage` handles this).
- Focus rings: 2px clay, 2px offset (`--ring: var(--color-clay)` — configured in Color section).
- Logout button has `aria-label="Log out of Barterkin"` since the visible label is just "Log out".
- Legal pages use proper heading hierarchy (single H1 → multiple H2 → H3 if needed; no skipping levels).
- Color contrast: sage-bg `#eef3e8` on forest `#2d5a27` = 8.2:1 (AAA). Clay `#c4956a` on sage-bg `#eef3e8` for button-chip-style = 3.4:1 — does NOT meet AA for body text (4.5:1) but DOES meet AA for UI components (3:1). Therefore clay is reserved for button backgrounds (with sage-bg foreground text = 6.9:1, AAA-pass) and focus rings, never body text.
