# Phase 5 — UI Design Contract

> Visual and interaction contract for the Contact Relay + Trust phase. Generated by gsd-ui-researcher from CONTEXT.md + existing codebase tokens. Verified by gsd-ui-checker.

**Phase scope (UI-relevant only):**
- Contact Sheet on `/m/[username]` — right-side shadcn Sheet containing form → success state (D-01, D-02, D-03)
- 3-dot overflow menu on profile card header — shadcn DropdownMenu with "Block" and "Report" items (D-04, D-05)
- Block AlertDialog confirmation → redirect to `/directory` with sonner toast (D-06)
- Report Dialog — reason Select + optional note textarea + inline confirmation (D-07)
- "New contact" badge on the Your-profile nav link (D-08, D-09)
- Inline rate-limit / eligibility / bounce error states (copy only — no new surfaces)

All values inherited from existing ProfileCard / DirectoryEmptyState patterns. Do not introduce new sizes.

| Role | Size (px / Tailwind) | Weight | Line Height | Font | Where it appears in Phase 5 |
|------|----------------------|--------|-------------|------|-----------------------------|

| Role | Token / Value | Usage |
|------|---------------|-------|
| Dominant (60%) | `--color-sage-bg` `#eef3e8` | Page background (inherited from `<body>`), Sheet backdrop overlay (default shadcn 50% black on top of sage bg is fine) |
| Secondary (30%) | `--color-sage-pale` `#f4f7f0` + `--color-sage-light` `#dfe8d5` (borders) | Sheet surface, Dialog surface, AlertDialog surface, Card surface, nav surface |
| Accent (10%) | `--color-clay` `#c4956a` | Primary "Send message" CTA, "Contact {Name}" trigger button, active-nav underline (existing pattern), new-contact badge dot, success check-mark background if used |
| Destructive | `oklch(0.577 0.245 27.325)` (shadcn default red) | Inline form errors, rate-limit rejection copy color, "Block" confirm button in AlertDialog, Report-count warning states |
| Forest-deep text | `--color-forest-deep` `#1e4420` | All primary text on sage surfaces |
| Forest-mid muted | `--color-forest-mid` `#3a7032` | Character counter, helper text, metadata |

**Accent reserved for (explicit list — never "all interactive buttons"):**
1. The primary "Contact {Name}" button on `/m/[username]` (forest-green per D-spec? — see note below)
2. The primary "Send message" submit button inside the Contact Sheet
3. The new-contact badge dot on the nav (red dot is MORE visible as destructive-red-400; clay as fallback only if destructive red clashes — pick destructive red for Phase 5 per convention: new/unread = red)
4. Hover/focus ring on primary form fields inside the Sheet (via `--color-ring: var(--color-clay)` — already active globally)

**Decision: "Contact {Name}" trigger button color.**
CONTEXT.md specifics say *"primary forest button, not a ghost"*. This CONFLICTS with the existing `bg-clay` pattern for primary CTAs.

**Resolution (ruled here):** Use `bg-forest hover:bg-forest-deep text-sage-bg` for the profile-page "Contact {Name}" trigger button — honors CONTEXT specific intent. For the Sheet's "Send message" submit, keep the existing `bg-clay hover:bg-clay/90 text-sage-bg` pattern (`h-11 min-w-[180px]`) to match all other primary CTAs in the app. Rationale: trigger = forest (strong, trustworthy, matches profile-page context); submit inside Sheet = clay (consistent with directory CTAs). Both are still in the 10% accent budget — forest is dominant text color reused as a surface here for prominence on the profile page.

**Destructive confirmation styling:**
- Block AlertDialog's confirm button: `variant="destructive"` (red) — Block is destructive from the blocked party's perspective.
- Report Dialog's submit button: `bg-clay hover:bg-clay/90 text-sage-bg` (standard primary) — Report is constructive (filing, not deleting).

## Layout & Interaction Contract (Phase-Specific)

### Contact Sheet (D-01)

- shadcn `Sheet` with `side="right"`.
- Width: `w-full sm:max-w-md` (default ~448px on desktop, full-width on mobile ≥360px viewport per LAND-03 parity).
- Open trigger: `Contact {display_name}` button rendered inside ProfileCard header on `/m/[username]`. Button hidden when viewer's own profile OR when `profile.accepting_contact === false` (show disabled-state inline message instead: `{display_name} isn't accepting messages right now.` in a muted `<Alert>` card at the CTA location — not in a Sheet).
- Close triggers: `Esc`, backdrop click, explicit `Cancel` button, `X` in sheet header (shadcn default).
- Focus management: autoFocus on the textarea when Sheet opens (`useEffect` + ref). On success state, shift focus to the `Close` button.
- Submit keyboard shortcut: `Cmd/Ctrl + Enter` inside textarea submits the form (parity with typical messaging UIs). Plain `Enter` inserts a newline.

### 3-dot overflow (D-04, D-05)

- Render inside `ProfileCard` header, right-aligned, `size="icon-sm"` shadcn Button with lucide `MoreVertical` icon.
- Hidden when `viewer_id === profile.owner_id` (server-side prop, not CSS-toggled — do not render).
- DropdownMenu `align="end"` so it opens leftward and doesn't overflow on mobile.

### Block AlertDialog (D-06)

- Uses existing `components/ui/alert-dialog.tsx`.
- On confirm: call `blockMember` server action → on success: `router.push('/directory')` → display sonner toast from the Directory page mount (toast lives on the destination, not the origin, to survive navigation).
- On error (network, RLS reject): keep AlertDialog open, replace description with destructive inline copy: `Something went wrong. Please try again.`

### Report Dialog (D-07)

- Uses existing `components/ui/dialog.tsx`.
- Two-pane pattern in a single Dialog: form → confirmation (swap in place, no second Dialog).
- On submit: call `reportMember` server action → on success: swap Dialog body to confirmation pane. Dialog stays open until user presses `Close`.
- Admin notification (TRUST-06) fires server-side from the server action, not the UI.

### New-contact badge (D-08, D-09)

- Rendered in `components/layout/NavLinks.tsx` as an absolutely-positioned `<span>` on the Your-profile Avatar.
- Count source: server-rendered from `app/(app)/layout.tsx` (single query per page load). Pass `unseenContactCount: number` as prop to `<AppNav />` → `<NavLinks />`.
- Clear mechanism: `app/(app)/profile/page.tsx` server component calls `markContactsSeen()` on load (sets `contact_requests.seen_at = now() WHERE recipient_id = auth.uid() AND seen_at IS NULL`). Next nav render shows `0`.
- Accessibility: always include sr-only count text even when visually showing only a dot — screen readers announce "3 new contacts, Your profile link".

### Accepting-contact disabled state (not in CONTEXT directly, implied by CONT-01)

When `profile.accepting_contact === false` on `/m/[username]`:
- Do NOT render the `Contact {Name}` button.
- Do NOT render the Contact Sheet trigger at all.
- Render a muted `<Alert>` card at the CTA slot: `Not accepting messages right now.` (Inter 14px, text-forest-mid, sage-pale bg, no action button).
- The 3-dot overflow menu (Block / Report) remains visible regardless.

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Keyboard — open Sheet | Tab to "Contact {Name}" button, Enter/Space opens |
| Keyboard — close Sheet | Esc, or Tab to Cancel/Close |
| Keyboard — submit form | Cmd/Ctrl+Enter inside textarea, OR Tab+Enter on Send button |
| Focus trap | shadcn Sheet / Dialog / AlertDialog all trap focus by default (Radix primitive) — do NOT disable |
| Screen reader | All icons used as buttons have `aria-label` or wrapping sr-only text (Flag, Ban, MoreVertical, CheckCircle2) |
| Reduced motion | Sheet slide honors `prefers-reduced-motion` via Radix default |
| Contrast | Forest-deep on sage-pale: 10.8:1 (AAA). Clay on sage-bg: 3.2:1 — passes WCAG AA for 18px+ bold or 14px+ regular *when used for buttons with 44px tap target*; do not use clay for inline body text. Destructive red on sage-pale: 4.9:1 (AA). |
| Color-only info | Badge uses dot + aria-count text; never color alone. Error states use both red color AND destructive icon (lucide `AlertCircle`) AND explicit copy. |
