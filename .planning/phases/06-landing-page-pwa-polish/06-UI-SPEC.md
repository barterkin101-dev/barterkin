## Spacing Scale

Declared values (all multiples of 4, inherited from Tailwind v4 defaults — no exceptions in this phase):

| Token | Value | Usage in Phase 6 |
|-------|-------|------------------|
| xs | 4px (`gap-1`) | Icon-to-text gaps inside buttons (e.g. Sprout icon + "Join the network") |
| sm | 8px (`gap-2`, `p-2`) | Pill padding, compact inline stacks (avatar + name on founding card) |
| md | 16px (`gap-4`, `p-4`, `space-y-4`) | Default inner spacing of CTA row, stat stack, how-card inner padding |
| lg | 24px (`gap-6`, `p-6`, `space-y-6`) | Container horizontal padding (`px-6`), founding-card `p-6`, county-card `p-6` — matches DirectoryCard convention |
| xl | 32px (`gap-8`, `p-8`, `space-y-8`) | Stat-strip gap on desktop, how-grid column gap on desktop |
| 2xl | 48px (`py-12`) | How-it-works header → grid gap; founding-strip header → row gap |
| 3xl | 64px (`py-16`, `mt-16`) | Landing section vertical rhythm (each major section gets `py-16` on mobile, `py-24` on desktop) |

**Section rhythm:** Each major section (Hero / Paths / How / Founding / Coverage / CTA) uses vertical padding `py-16 sm:py-20 md:py-24`. The hero is the only exception: `pt-20 pb-24 sm:pt-28 sm:pb-32` (slightly taller to earn the gradient).

**Exceptions:**
- Hero vertical padding (`pt-20 pb-24 sm:pt-28 sm:pb-32`) — these are all multiples of 4 but larger than the standard 3xl token. Ratified here, not a drift.
- All tap targets ≥44px (`h-11` minimum on CTAs; `h-14` permitted on the hero primary CTA for visual weight). Matches LAND-03 success criterion 3.

## Typography

Declared exactly 4 sizes + 2 weights. All inherited from existing ProfileCard / DirectoryEmptyState / legacy/index.html conventions. Do not introduce new sizes.

| Role | Size (px / Tailwind) | Weight | Line Height | Font | Where it appears on the landing page |
|------|----------------------|--------|-------------|------|---------------------------------------|
| Body | 16px (`text-base`) | 400 (Inter regular) | 1.5 (`leading-[1.5]`) | Inter sans | Hero sub, section sub-headers, how-card body, founding-card meta, footer copy |
| Label | 14px (`text-sm`) | 500 (Inter medium via shadcn `<Badge>` / `<Button>` defaults) | 1.5 | Inter sans | Section eyebrow, stat labels, category chip, county chip, CTA button text, nav link |
| Heading | 24px (`text-2xl`) on mobile / 32px (`text-3xl`) on desktop | 700 bold | 1.2 (`leading-[1.2]`) | Lora serif (`font-serif`) | Section titles ("How it works", "Meet our founding members", "Counties we cover"), how-card step heading, founding-card name heading |
| Display | 40px (`text-4xl`) on mobile / 64px (`text-6xl`) on desktop | 700 bold | 1.1 (`leading-[1.1]`) | Lora serif (`font-serif`) | Hero h1 only — single use |

**Fluid hero headline:** `text-4xl sm:text-5xl md:text-6xl` (40 / 48 / 60 / 64 via Tailwind's default scale). No `clamp()` — the legacy index.html used clamp, but Tailwind v4 step scale achieves the same effect with fewer moving parts.

**Heading color:**
- On sage surfaces: `text-forest-deep` (`#1e4420`) always.
- On forest gradient (hero): `text-sage-bg` (`#eef3e8`) — near-white for contrast without pure `#fff` (avoids harsh edge against the warm palette).

**Body color:**
- On sage surfaces: `text-forest-deep` for primary copy, `text-forest-mid` (`#3a7032`) for meta / sub-labels only.
- On forest gradient (hero): `text-sage-bg/80` for the sub-headline (80% opacity mimics legacy's `rgba(255,255,255,0.8)`).

**Section eyebrow:** `text-xs uppercase tracking-[0.15em] font-semibold text-forest-mid` — mirrors legacy's `.section-eyebrow` style but replaces the teal accent with `forest-mid` (we do NOT introduce teal — see Color §).

**Font-weight count:** 2 weights used — 400 (Inter regular) and 700 (Lora bold). Inter 500 is used by shadcn `<Button>` / `<Badge>` / `<Label>` internally; accepted as a system-layer default, not a new declared weight.

## Color

| Role | Token / Value | Usage |
|------|---------------|-------|
| Dominant (60%) | `--color-sage-bg` `#eef3e8` | Page background, how-it-works section, county-coverage section, page body default |
| Secondary (30%) | `--color-sage-pale` `#f4f7f0` + `--color-sage-light` `#dfe8d5` (borders) | Founding-member card surface, how-card surface (`bg-white` is NOT used — we stay sage), county-chip surface |
| Tertiary surface (<10%) | `--color-forest-deep` → `--color-forest` gradient | Hero section background only (`bg-gradient-to-b from-forest-deep via-forest to-forest-mid`). Single location. |
| Accent (10%) | `--color-clay` `#c4956a` | Primary CTA bg ("Join the network", "Get started"); active pagination dot color is out of scope here |
| Destructive | `oklch(0.577 0.245 27.325)` (shadcn default red) | Not used on landing page. Reserved for authed app surfaces only (Phase 5). |
| Forest-deep text | `--color-forest-deep` `#1e4420` | All primary text on sage surfaces |
| Forest-mid muted | `--color-forest-mid` `#3a7032` | Stat labels, section eyebrows, meta, founding-card "County · Category" line |

**Deliberate deviation from legacy/index.html:** The legacy page used a `--teal` accent (`#2a9d8f`) for secondary CTAs + eyebrows + form-submit. We do NOT port teal into the Next.js app. Reasons:

1. Phase 1's `app/globals.css` only ported six tokens (sage-bg/light/pale, forest/deep/mid, clay). Teal was intentionally omitted per FOUND-02.
2. Phases 2–5 have shipped without teal. Introducing it here breaks palette consistency.
3. Clay already serves as the single accent (`--color-primary`); teal would create a two-accent split that violates 60/30/10.

Anywhere the legacy used teal (hero primary CTA, eyebrow label color, form focus ring), we substitute:
- **Hero primary CTA**: clay (`bg-primary text-primary-foreground`, which resolves to clay on sage).
- **Section eyebrow text color**: `forest-mid` (muted green), not teal. Legacy teal served as visual accent; forest-mid integrates with our palette.
- **Focus ring**: clay (already globally assigned via `--color-ring`).
- **Footer link underline**: clay on hover (already in Footer.tsx).

**Accent reserved for (explicit list — never "all interactive buttons"):**
1. Hero primary CTA ("Join the network") — `bg-clay hover:bg-clay/90 text-sage-bg` (matches the Send-message button pattern from Phase 5)
2. Secondary-section primary CTA ("Join the network" in the bottom CTA strip) — same styling
3. Footer legal link underline on hover (already in Footer.tsx)
4. Focus ring on interactive elements (already globally via `--color-ring`)

**Hero secondary CTA** ("Browse the directory") uses `variant="outline"` with white border + transparent bg against the forest gradient — NOT clay. It is a secondary action and must not share accent.

**How-card step numbers** ("1", "2", "3") use `bg-forest text-sage-bg` (forest as the step-circle fill) — mirrors legacy without introducing a new accent.

**Founding-member card** uses `bg-sage-pale ring-1 ring-sage-light` — IDENTICAL to DirectoryCard to preserve parity when users move landing → directory.

### Page-level CTA ordering rule

The primary CTA is always `Join the network` / `Join Barterkin` (anonymized wording for anon viewers). ### Error state (LAND-02 — if founding-member or county queries fail)

The landing page never renders a server-error screen for the signed-out marketing visitor. If either the founding-member or county-coverage query fails:

| Scenario | Fallback |
|----------|----------|
| Founding-member query fails | Render the empty-state copy above (`Be a founding member. …`). Log to PostHog as `landing_founding_strip_error`. |
| County-coverage query fails | Render the empty-state copy above (`No counties yet. Yours could be first.`). Log to PostHog as `landing_county_coverage_error`. |
| Stat-strip query fails | Render the fallback values (`30+`, `2+`, `10`). Log to PostHog as `landing_stat_strip_error`. |

**Never** show a user-facing error page. This is the public marketing front door — a broken section must degrade gracefully, not 500.

### Destructive confirmations

None in this phase. The landing page has zero destructive actions (no sign-out, no delete, no block). All primary actions are navigational.

## Responsive & Mobile Contract

Mobile-first. LAND-03 pins the floor at 360px viewport (iPhone SE 1st gen width).

| Surface | Mobile (<640px) | Tablet (640–1024px) | Desktop (≥1024px) |
|---------|-----------------|---------------------|-------------------|
| LandingNav | brand + "Join" CTA only | brand + links + CTA | brand + links + CTA |
| Hero h1 | `text-4xl` (40px), centered | `text-5xl` (48px), centered | `text-6xl` (60–64px), left-aligned |
| Hero CTAs | stacked vertical, full-width `h-14` | inline row `h-12` | inline row `h-12` |
| Hero stats | stacked vertical `gap-6` | 3-col row `gap-10` | 3-col row `gap-14` |
| How grid | 1 col `gap-4` | 3 col `gap-4` | 3 col `gap-6` |
| Founding strip | 1 col `gap-4` | 2 col `gap-4` | 3 col `gap-6` |
| County pills | wrap on 2–3 rows | wrap on 2 rows | wrap on 1–2 rows |
| SecondaryCTA | centered, full-width CTA `h-14` | centered, auto-width CTA `h-14` | centered, auto-width `h-14` |
| Footer (existing) | 1 col stacked | 3 col | 3 col |

**Tap target floor:** All CTAs use `h-11` (44px) minimum. Hero primary uses `h-14` (56px) on mobile for thumb-comfort; desktop reduces to `h-12` (48px) since mouse pointing is more precise. Outline/ghost buttons also respect `h-11`.

**Horizontal overflow check:** At 360px viewport, every section must fit without horizontal scroll. Verified via Playwright's `mobile` project in existing test infra (Phase 1 wired `devices.iPhoneSE`).

**Text legibility:** Minimum body text size is 14px (`text-sm`). Hero sub-headline is 16px. H1 at 40px on mobile is the smallest display — legible without zoom per LAND-03.

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Keyboard — tab order | Brand → nav links → primary CTA → secondary CTA → hero primary → hero secondary → stats (non-tabbable) → how-cards (not interactive, not tabbable) → founding cards (each is a `<Link>`, tabbable) → browse-all CTA → county pills (not interactive) → secondary-CTA primary → guidelines link → footer links |
| Keyboard — skip link | Add `<a href="#main" className="sr-only focus:not-sr-only focus:fixed focus:top-2 focus:left-2 focus:z-50 focus:bg-forest focus:text-sage-bg focus:px-4 focus:py-2 focus:rounded">Skip to content</a>` at the top of `<body>` — applies to the whole site, add to `app/layout.tsx` |
| Screen reader — hero | H1 is the page's only h1. Hero eyebrow pill has `role="note"` (not a tab stop). Stats use `<dl>` with `<dt>` label + `<dd>` value for proper semantics |
| Screen reader — how steps | Each step card's "1"/"2"/"3" is inside an `aria-hidden="true"` decorative `<div>` — the heading ("List what you offer") carries the semantic weight |
| Screen reader — founding card | Link `aria-label` follows the DirectoryCard pattern: `View {name}'s profile — {county} County — {category} — offers {skill1}, {skill2}, {skill3}` |
| Screen reader — county pills | Container has `aria-label="Counties with members on Barterkin"`, pills are plain `<span>` (non-interactive, descriptive text only) |
| Reduced motion | Hover-lift on founding cards (`hover:-translate-y-0.5`) respects `motion-reduce:hover:translate-y-0` — matches DirectoryCard convention. No parallax, no scroll-triggered animations on this landing |
| Contrast | Forest-deep on sage-pale: 10.8:1 (AAA). Sage-bg on forest-deep (hero text on gradient): 12.4:1 (AAA). Clay on sage-bg (CTA): 3.2:1 — passes WCAG AA for 18px+ bold text AND for CTAs where `h-14` tap target provides adequate visual weight; body-text contrast would fail, so we never use clay for inline text |
| Color-only info | Stat strip numbers + labels are both present (no color-coding); CTAs have distinct shape (filled vs outline) not just color; eyebrow labels use uppercase letterspacing for typographic differentiation alongside color |
| Focus ring | All buttons + links get the global `--ring: var(--color-clay)` ring via shadcn's default `focus-visible:ring-2 focus-visible:ring-ring` |
| Language | `<html lang="en">` already set in `app/layout.tsx` |
| Viewport | `viewport` exported from `app/layout.tsx` includes `themeColor: '#2d5a27'` and `width: 'device-width'` — already correct |

## Out-of-Spec Guardrails

If during execution the planner or executor feels the need to do any of the following, they MUST pause and amend this spec before proceeding:

- Introduce teal (`#2a9d8f`) or any new hue
- Add a new font weight (500 from shadcn defaults is grandfathered; 300/600/800 are not)
- Add a font size outside 14 / 16 / 24 / 32 / 40 / 48 / 60 / 64
- Install a new shadcn primitive not already in `components/ui/`
- Install anything from a third-party registry
- Embed a sign-up form directly on the landing page (we link to `/signup`, we don't duplicate it)
- Add photography or stock imagery (the brand is type + palette + illustration; no photos in v1)
- Add scroll-triggered animations, parallax, or video backgrounds
- Ship a custom `beforeinstallprompt` handler (rely on browser default for v1)
- Change footer copy or structure (Footer is cross-page; changes go through a separate phase)
