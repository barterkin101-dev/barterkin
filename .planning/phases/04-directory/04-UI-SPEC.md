# Phase 4 — UI Design Contract

> Visual and interaction contract for the Directory phase. Covers one primary surface — `/directory` — plus the profile-card grid unit, the filter bar (category + county + keyword), pagination control, empty-state and zero-results panels, and the in-nav "Directory" link promotion. Consumes the Phase 3 palette, typography, spacing, and shadcn new-york stone preset unchanged. No new design tokens introduced. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Spacing Scale

All values are multiples of 4, consumed from Tailwind v4's default 4px baseline (`1 = 4px`, `2 = 8px`, `4 = 16px`, `6 = 24px`, `8 = 32px`, `12 = 48px`, `16 = 64px`). Identical to Phase 3.

| Token | Value | Usage in Phase 4 |
|-------|-------|------------------|
| xs | 4px (`gap-1`, `p-1`) | Icon ↔ label gap inside filter chips, "showing X of Y" counter internal padding, badge dot spacing |
| sm | 8px (`gap-2`, `p-2`) | Skill-chip horizontal padding inside a card, filter-bar element internal spacing, pagination button internal icon-label gap |
| md | 16px (`gap-4`, `p-4`, `space-y-4`) | Card internal padding (between avatar row and skills row), gap between cards in a single column (mobile), gap between filter bar elements horizontally (desktop), gap between result cards vertically when grid collapses to one column |
| lg | 24px (`gap-6`, `p-6`, `space-y-6`) | Card internal padding desktop, gap between cards in the grid on `md+`, filter bar vertical padding on mobile (when stacked), heading ↔ filter-bar gap, filter-bar ↔ grid gap |
| xl | 32px (`gap-8`, `space-y-8`) | Page heading block ↔ filter bar gap, grid ↔ pagination gap, zero-results panel internal padding |
| 2xl | 48px (`gap-12`, `py-12`) | Mobile page top/bottom padding (matches Phase 3 `(app)` shell) |
| 3xl | 64px (`gap-16`, `py-16`) | Desktop page top/bottom padding (matches Phase 3 `(app)` shell) |

**Exceptions for Phase 4:**
- **Tap targets ≥ 44px**: all filter-bar controls (category trigger, county Combobox trigger, keyword input, clear-filters ghost button, active-filter chip × remove) render at `h-11` (44px) minimum. Pagination buttons (Previous / Next / page-number) render at `h-11 min-w-[44px]`. Card body is a single `<Link>` wrapper — the entire card is the 44×44-exceeding tap target (no nested tappable controls inside).
- **Page max-width**: the `(app)` shell's `<main>` currently uses `max-w-2xl` (672px) which is too narrow for a card grid. **Phase 4 changes `(app)/layout.tsx` `<main>` to `max-w-5xl` (1024px)** to accommodate the 1-column (mobile) / 2-column (`md`) / 3-column (`lg+`) card grid and the wide filter bar. This override is the only layout-shell change in Phase 4; `/profile`, `/profile/edit`, and `/m/[username]` remain visually centered because their inner `<Card>` still uses `max-w-2xl` / `max-w-3xl` internally.
- **Profile card dimensions**: fixed aspect isn't required, but each card uses `min-h-[220px]` and a vertical internal layout: avatar-64 + display-name + county/category line + top-3 skill chips. Avatar is 64px round (`h-16 w-16`) — smaller than the 128px editor preview and the 160px `/m/[username]` hero, consistent with a list-density surface.
- **Skill chips inside card**: `h-7` (28px) with `px-2.5` (10px horizontal), `gap-1` (4px) between chips. 28px is below the 44px touch floor, but chips inside a card are **display-only and non-interactive** (clicking anywhere in the card navigates to `/m/[username]`); no tap-target rule applies.
- **Filter bar height**: `h-12` (48px) on desktop as a single-row flex container; on mobile stacks vertically with `space-y-3` (12px) between filters. 12px is NOT on the 4-multiple scale — **mobile filter-bar inter-element gap overrides to `space-y-2` (8px)** to stay on the scale. Desktop retains the single 48px-tall row with `gap-3` between elements — **desktop filter gap overrides to `gap-2` (8px)** for the same reason. (Both corrections applied below in Visuals.)
- **Active-filter chip row** (appears below filter bar when ≥1 filter is active): `h-9` per chip (36px), `gap-2` (8px) between chips, `py-2` top/bottom padding. The row itself uses `py-4` (16px) separating it from the filter-bar and the grid.
- **Grid gutters**: `gap-4` (16px) on mobile, `gap-6` (24px) `md+`. No exceptions.
- **Pagination block**: `h-12` (48px) row, `gap-2` (8px) between buttons, `py-8` (32px) top spacing above, `pb-12 md:pb-16` matching shell bottom padding.

## Typography

Consumes the 2-weight rule unchanged from Phase 2/3: **400 (regular) and 700 (bold)**. Lora for headings, Inter for everything else.

| Role | Size | Weight | Line Height | Font | Phase 4 Surfaces |
|------|------|--------|-------------|------|------------------|
| Body | 16px (`text-base`) | 400 (`font-normal`) | 1.5 (`leading-[1.5]`) | Inter | Page sub-lede, filter-bar input text (keyword search), Combobox selected-county label, empty/zero-results body copy, card display-name (at 16px bold — see below) |
| Label | 14px (`text-sm`) | 400 (`font-normal`) | 1.4 (`leading-[1.4]`) | Inter | Filter-bar labels ("Category", "County", "Search"), active-filter chip text, card county · category sub-line, card skill-chip text, pagination button labels, "showing 1–20 of 47" counter, zero-results helper text |
| Heading | 24px (`text-2xl`) | 700 (`font-bold`) | 1.2 (`leading-[1.2]`) | Lora (`font-serif`) | Empty-state heading, zero-results heading, section heads if any. **Card display-name exception: card titles are 16px Inter bold (`text-base font-bold`)** — NOT 24px Lora — to keep card density tight and give the `/m/[username]` hero room to breathe at 24px. Cards are a listing unit, not a hero. |
| Display | 32px desktop / 28px mobile (`text-3xl md:text-[32px]`) | 700 (`font-bold`) | 1.15 (`leading-[1.15]`) | Lora (`font-serif`) | `/directory` page title `Directory` only |

**No other sizes.** No 12px microcopy. No `font-medium` / `font-semibold`. No italics anywhere in Phase 4 (Phase 3's italic-placeholder exception on skill rows does not appear on the directory surface).

**Responsive:**
- Display drops 32 → 28px below `md` (768px)
- Heading stays 24px at all widths
- Body never drops below 16px (iOS auto-zoom trigger floor)
- Card display-name stays at 16px at all widths

**Counter text** ("showing 1–20 of 47", "Page 1 of 3"): 14px Inter regular, `--color-muted-foreground` (forest-mid). Right-aligned at the top of the grid on `md+`, center-aligned on mobile. Counter goes `--color-destructive` only if the backend surfaces an error count (not expected in MVP — fallback for zero-results instead).

**Single-word truncation rule:** card display-names use `truncate` (single-line, ellipsis). Skill-chips inside a card truncate at 24 characters with an ellipsis if the skill text (max 60 per Phase 3) exceeds; the card is the ellipsized representation, not the detail view.

## Color

60/30/10 split inherits from Phase 2/3 exactly. The directory surface is information-dense; it leans **harder** on the 60% dominant than Phase 3 did, with cards providing the 30% secondary and clay accent reserved for the filter-bar "Apply" affordance, active-filter state, and the Next/Previous pagination buttons.

| Role | Value | Usage in Phase 4 |
|------|-------|------------------|
| Dominant (60%) | `--color-sage-bg #eef3e8` | `/directory` page background, grid-gutter background between cards, pagination row background, empty/zero-results outer panel background, card-hover backdrop ring (1px sage-light → no color shift, just ring) |
| Secondary (30%) | `--color-sage-pale #f4f7f0` | Profile card background (1px `--color-sage-light` border, 8px `rounded-lg`), filter-bar background (1px sage-light border, 8px rounded), County Combobox popover background, active-filter-chip background, empty/zero-results inner card background, pagination button background (outline variant), skill-chip background inside cards (`bg-sage-bg` on a sage-pale card for subtle contrast — same inversion Phase 3 `/m/[username]` uses) |
| Accent (10%) | `--color-clay #c4956a` | **Filter actions and navigation only** (see reserved-for list); focus rings on filter inputs (inherited `--color-ring`); the "Next page" and "Previous page" pagination buttons when enabled; active-filter-chip ring |
| Destructive | `oklch(0.577 0.245 27.325)` (shadcn default) | NOT USED in Phase 4. Zero-results state uses muted/forest-mid, NOT destructive. Destructive only renders if a real error boundary fires (e.g. Supabase query fails) — see Error states in Copywriting. |

**Accent reserved-for list (exhaustive for Phase 4):**
1. **Filter "Clear filters" button** — rendered as a ghost-link (14px forest-mid) when inactive; promotes to clay ghost-link underline on hover and when ≥1 filter is active. (Actual button fill: transparent; accent is the text color only.)
2. **Pagination "Next page" button when enabled** — clay bg + sage-bg text, `size="lg"`, `h-11`, with `ArrowRight` lucide icon 16px inline-right.
3. **Pagination "Previous page" button when enabled** — clay bg + sage-bg text, `size="lg"`, `h-11`, with `ArrowLeft` lucide icon 16px inline-left.
4. **Active-filter chip ring** — each active filter (e.g. "Category: Arts & Crafts", "County: Fulton County", "Search: baking") renders as a `Badge variant="secondary"` with `bg-sage-pale` + `ring-1 ring-clay` + forest-deep text + `X` lucide icon 14px inline-right as the remove affordance. The clay ring is the only accent on the chip; the chip background stays sage-pale.
5. **Empty-state / zero-results primary CTA** — clay bg, `size="lg"`, `h-11`. Empty state CTA routes to `/profile/edit`; zero-results CTA clears filters.
6. **Focus ring on filter inputs** — 2px clay, 2px offset (Phase 2 `--color-ring` inherited).
7. **County Combobox selected-row highlight** — `data-[selected=true]:bg-clay data-[selected=true]:text-primary-foreground`, inherited from Phase 3 unchanged.
8. **Category dropdown selected-option highlight** — same `bg-clay text-primary-foreground` pattern inside the Command list.
9. **In-nav "Directory" link active state** — when the user is on `/directory`, the "Directory" nav link switches from `text-forest-mid` to `text-forest-deep` with a 2px clay underline (`border-b-2 border-clay pb-1`). Non-active nav links remain unchanged from Phase 3.

**NOT allowed for accent in Phase 4:**
- Profile card background (sage-pale only)
- Profile card border (sage-light only; 1px)
- Profile card hover state — hover goes to `ring-1 ring-sage-light` (slightly darker) and `shadow-sm`; **no clay on hover**. Cards are directory rows, not CTAs.
- Avatar ring inside cards (1px sage-light, no clay)
- Skill chips inside cards (sage-bg background, forest-deep text — identical to `/m/[username]` chips)
- County / category filter-bar triggers when NOT dropdown-open (outline-variant buttons with sage-light border, forest-deep text; clay appears only on focus-ring and the dropdown's selected-item highlight, never on the closed trigger)
- Counter text "showing 1–20 of 47" (muted-foreground only)
- Pagination buttons when DISABLED (outline variant with muted text; no clay fill)
- Any card text (display-name uses forest-deep; county · category uses forest-mid; skill-chip text uses forest-deep; NO clay text anywhere on a card)

**Hover states recap:**
- Profile card: `hover:ring-1 hover:ring-sage-light hover:shadow-sm hover:-translate-y-0.5` — subtle lift, no color change, respects `prefers-reduced-motion` via `motion-reduce:hover:translate-y-0`.
- Filter triggers: `hover:bg-sage-bg` (darker sage background on the sage-pale filter bar).
- Pagination enabled: `hover:bg-clay/90` (clay darkens slightly on hover).
- Pagination disabled: no hover effect.
- "Directory" nav link (on other pages): `hover:text-forest-deep` (inherited Phase 3 pattern).
- Active filter chip × remove: `hover:text-destructive` — the ONLY destructive usage in Phase 4, and it's a foreground color on hover only. Chip background and ring stay unchanged on hover.

**Logical color rule:** Phase 4 introduces **zero** new named colors. All surfaces consume the eight existing CSS vars. Raw hex, new Tailwind arbitrary-color classes (`bg-[#...]`), or fresh `--color-*` declarations fail this contract.

## Accessibility Contract

- **Keyboard navigation (visual order = tab order):**
  1. Nav (inherited Phase 3)
  2. Category filter trigger
  3. County filter trigger
  4. Keyword search input
  5. Clear filters button (if visible)
  6. Each active-filter chip's remove × (if chips visible)
  7. Each profile card `<Link>` (top-to-bottom, left-to-right through the grid)
  8. Previous pagination button
  9. Next pagination button
- **Focus rings**: 2px clay, 2px offset — on ALL interactive elements (filters, chip-remove, card links, pagination, clear-filters link). Inherited from Phase 2 `--color-ring`.
- **Card link semantics**: each card is a single `<Link>` wrapping the entire card surface. No nested interactive elements. `aria-label` describes the card content for screen readers (see Copywriting).
- **Filter controls**:
  - Category trigger: shadcn `Popover` + `Command` default ARIA (listbox + option roles); `aria-expanded` on trigger.
  - County trigger: identical to Phase 3.
  - Keyword input: `<input type="search" role="searchbox" aria-label="Search by keyword">`. The visible placeholder is supplementary; the aria-label is the authoritative accessible name.
- **Active-filter chip remove button**: `aria-label="Remove {field} filter"` since the visible label is a `×` icon.
- **Pagination**:
  - Previous/Next: `aria-label` as specified in Copywriting; `aria-disabled="true"` + `tabindex="-1"` when disabled (non-focusable). Counter is a `<span>` with `role="status" aria-live="polite"` so screen readers announce page changes.
- **Loading state**: outer grid wrapper has `role="status" aria-label="Loading directory results"`; each skeleton card is `aria-hidden="true"` (screen readers hear the status, not the visual placeholders).
- **Empty / zero-results / error states**: outer wrapper uses `role="alert"` for the zero-results and error panels (so screen readers announce on state change without user re-navigating). Empty state on initial load uses plain region — no alert — since it's not a state change from a user action.
- **Color contrast (verified against Phase 3 baseline):**
  - Forest-deep `#1e4420` on sage-pale `#f4f7f0` (card display-name, body) = 10.2:1 (AAA)
  - Forest-mid `#3a7032` on sage-pale (county · category line, muted-foreground, counter) = 5.4:1 (AA pass for 14px, AAA fail — acceptable at 14px+)
  - Clay `#c4956a` bg on sage-bg `#eef3e8` foreground (pagination buttons) = 6.9:1 (AAA)
  - Clay `#c4956a` 2px ring on sage-pale bg (active-filter chip) — ring is decorative; chip text contrast is forest-deep on sage-pale (10.2:1 AAA)
  - Destructive (shadcn default oklch) on sage-pale (chip-remove hover, keyword-clear hover, error icon) = 4.7:1 (AA pass for body text)
- **Touch targets ≥ 44px**: all filter triggers, keyword input, chip-remove, pagination buttons render at `h-11` minimum. Card link is the entire card (≥220px tall, ≥280px wide on mobile).
- **Reduced motion**: card hover lift (`-translate-y-0.5`) respects `prefers-reduced-motion` via `motion-reduce:hover:translate-y-0 motion-reduce:transition-none`. Skeleton `animate-pulse` respects `motion-reduce:animate-none`. shadcn defaults (Combobox open/close, toast enter/exit) honor reduced-motion via Radix — no extra work.
- **Skip-link**: not added in Phase 4 (shell is short; revisit in Phase 6 if landing page grows).
- **URL-shareability ↔ accessibility**: filter state mirrors in the URL via `?category=X&county=Y&q=Z&page=N` (DIR-06); pasting the URL in another browser restores both the filter UI state AND the announced page state (because SSR re-renders with the same filters). No history-stack corruption — each filter change pushes a new URL via `router.push` (NOT `router.replace`) so back-button navigation returns to the previous filter state.
