> Visual and interaction contract for the Profile & Georgia Gate phase. Covers 2 primary surfaces: `/profile/edit` (single-page scrollable editor) and `/m/[username]` (member profile view, auth-gated), plus the `/profile` own-view with publish toggle, the `(app)` authenticated shell layout, and inline components (avatar uploader, skills row list, county combobox, publish gate, success toast). Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Spacing Scale

All values in multiples of 4. Tailwind v4 default 4px baseline applies (`1 = 4px`, `2 = 8px`, `4 = 16px`, `6 = 24px`, `8 = 32px`, `12 = 48px`, `16 = 64px`).

| Token | Value | Usage in Phase 3 |
|-------|-------|------------------|
| xs | 4px (`gap-1`, `p-1`) | Avatar crop preview ‚Üí filename label gap; icon-label gap inside "+ Add skill" button; inline counter ("3/5") padding |
| sm | 8px (`gap-2`, `p-2`) | Skill-row internal spacing (input ‚ÜĒ remove-√ó); checkbox/label horizontal gap; helper text top-margin below inputs; form-section intro subhead ‚ÜĒ body gap |
| md | 16px (`gap-4`, `p-4`, `space-y-4`) | Default form-field vertical rhythm (label ‚Üí input ‚Üí help text); two adjacent section labels; card internal padding on `/m/[username]` skill chips |
| lg | 24px (`gap-6`, `p-6`, `space-y-6`) | Section internal padding inside the editor card; vertical rhythm between sibling fields within a section; avatar-upload module internal padding |
| xl | 32px (`gap-8`, `space-y-8`) | Vertical gap **between sections** on `/profile/edit` (Basic Info ‚Üí Skills I Offer ‚Üí Skills I Want ‚Üí Location & Category ‚Üí Preferences). Also: top-gap between `/m/[username]` hero and skill grid. |
| 2xl | 48px (`gap-12`, `py-12`) | Mobile page padding top/bottom on `/profile/edit` and `/m/[username]`; gap between last section and sticky Save bar |
| 3xl | 64px (`gap-16`, `py-16`) | Desktop page padding top/bottom on `/profile/edit` and `/m/[username]`; major visual break between form card and any supporting helper card |

**Exceptions for Phase 3:**
- **Tap targets ‚Č• 44px**: "+ Add skill" button, remove-√ó button on each skill row, avatar-upload button, publish toggle, Save button all render at `h-11` (44px) minimum. shadcn `Button size="lg"` (`h-10` = 40px) is **not sufficient** on touch ‚ÄĒ Phase 3 overrides to `h-11` where tap-only targets appear. Skill `√ó` remove uses `size="icon-sm"` (32px visual) but gets a 44√ó44px expanded hit area via an outer wrapper `<button aria-label="Remove skill" class="flex h-11 w-11 items-center justify-center">`.
- **Editor form max-width**: `max-w-2xl` (672px) ‚ÄĒ longer than auth card (400px) because the editor has multi-field rows (skill input + remove button side-by-side) and benefits from room to breathe. On `/m/[username]` the content max-width is `max-w-3xl` (768px) for the hero + 2-column skills grid.
- **Avatar dimensions (rendered)**: 128px on `/profile/edit` upload preview, 160px on `/m/[username]` hero, 64px on future directory cards (Phase 4 reference). All square (1:1), rendered via shadcn `Avatar` with `rounded-full`.
- **Skill chip on `/m/[username]`**: `h-8` (32px) with `px-3` (12px horizontal), `gap-1` (4px) between text and lucide icon. 32px is below the 44px touch floor but chips are **display-only, non-interactive** ‚ÄĒ no tap-target rule applies.
- **Sticky Save bar (mobile)**: height `h-16` (64px) with `px-6 py-4` internal padding; contains Save button (`h-11`) + "saved just now" status text inline. On desktop (‚Č• `md`) the Save button lives at the bottom of the card and the sticky bar is hidden.
- **County Combobox dropdown**: Popover content `max-h-[320px]` (80 √ó 4 = 320, divisible by 4) with per-item `h-9` rows. 159 counties render as a virtualized `Command` list; sticky search input at top (`h-10` = 40px).

## Typography

Consumes the 2-weight Phase 2 rule unchanged: **400 (regular) and 700 (bold)**. Lora renders headings; Inter renders everything else.

| Role | Size | Weight | Line Height | Font | Phase 3 Surfaces |
|------|------|--------|-------------|------|------------------|
| Body | 16px (`text-base`) | 400 (`font-normal`) | 1.5 (`leading-[1.5]`) | Inter | Bio textarea, skill row inputs, availability textarea, help text under fields, empty-state body copy, `/m/[username]` bio paragraph |
| Label | 14px (`text-sm`) | 400 (`font-normal`) | 1.4 (`leading-[1.4]`) | Inter | Form field labels ("Display name", "Bio", "Skills I offer", "County", "Primary category", "Availability", "TikTok handle"), character counters ("42/500"), section sub-labels, skill-chip text on `/m/[username]` |
| Heading | 24px (`text-2xl`) | 700 (`font-bold`) | 1.2 (`leading-[1.2]`) | Lora (`font-serif`) | Editor page heading ("Your profile"), each section heading on `/profile/edit` ("Basic info", "Skills I offer", "Skills I want", "Location & category", "Preferences"), `/m/[username]` member display-name heading |
| Display | 32px desktop / 28px mobile (`text-3xl md:text-[32px]`) | 700 (`font-bold`) | 1.15 (`leading-[1.15]`) | Lora (`font-serif`) | `/profile/edit` page title only when rendered as a top-level page header (not sub-sectional). Also: `/m/[username]` display-name heading when the member has no avatar ‚ÄĒ display-name promotes to 32px as compensation. |

**No other sizes.** No 12px microcopy in Phase 3 (the smallest is 14px label). No `font-medium` / `font-semibold`. No italics except for placeholder text (`placeholder:italic` on skill rows intentionally to signal "example, not current value").

**Responsive:**
- Display drops 32 ‚Üí 28px below `md` (768px)
- Heading stays 24px at all widths
- Body never drops below 16px (iOS auto-zoom trigger floor) ‚ÄĒ all textarea + input use `text-base`

**Character counters** (e.g. `42/500` for bio, `12/60` for skill, `8/200` for availability) render at 14px Inter regular in `--muted-foreground`, right-aligned below the field. Counter flips to `--destructive` at 100% of max.

## Visuals

**Focal point declaration per surface.**

**`/profile/edit` (editor):** The primary eye path is page title (Lora 32px bold) ‚Üí first section heading "Basic info" (Lora 24px bold) ‚Üí display-name input (top field, always the first tab-stop) ‚Üí down through sections. The clay-background "Save profile" button at the bottom of the card is the secondary focal point and anchors the page ‚ÄĒ every section reads toward it. On mobile, the sticky bottom Save bar keeps the accent always in view, but the inline Save button at card-bottom is the canonical end-of-form action.

**`/m/[username]` (member view):** The focal point is the avatar (160px round, 1px sage-light border) + display-name (Lora 24px bold, or 32px if no avatar) in the hero band. Secondary anchors: primary-category badge and county line directly below the name. Skills grid sits below the hero with an xl (32px) vertical gap. No clay accent appears above the hero ‚ÄĒ the page communicates "this is a person, not a marketplace listing."

**`/profile` (own view):** Identical to `/m/[username]` visually, plus a clay "Edit profile" CTA in the top-right of the hero, plus the publish toggle (Switch) in its own inline row below the hero. Publish toggle shows inline state ("Published" in forest / "Unpublished" in muted) to the right of the switch.

**Section dividers on `/profile/edit`:** A single `<Separator>` (1px `--color-sage-light`) separates each of the 5 sections. The separator is NOT clay ‚ÄĒ it's a neutral rhythm element. Clay is reserved for CTAs (see reserved-for list).

**Publish-gate tooltip UI (when the user hovers/focuses the disabled publish toggle):** Renders a shadcn `Tooltip` with a sage-pale background, 1px sage-light border, 6px rounded, containing a bulleted checklist:
```
Profile must be complete to publish:
  ‚úó Display name
  ‚úó County
  ‚úó Primary category
  ‚úď At least one skill offered
  ‚úó Avatar
```
Checkmarks use `lucide-react` `Check` in forest-mid for met items; crosses use `X` in destructive for unmet items. The tooltip auto-positions `side="top"` and has `max-w-[300px]`.

**Sticky mobile Save bar:** Shows only below `md` breakpoint. Position `fixed bottom-0 left-0 right-0`, background `bg-sage-pale` with `border-t border-sage-light`, `h-16`, `px-6 py-3`, flex row `justify-between`: left shows "Unsaved changes" in 14px forest-mid OR "Saved" in 14px muted depending on RHF `formState.isDirty`; right shows clay Save button `size="lg"` full-height. When form is pristine, the bar is hidden (`formState.isDirty === false`).

**Avatar upload affordance:** Replace the standard file-input chrome with a custom drop-zone: 128√ó128 rounded-full placeholder with dashed `border-2 border-sage-light` when empty, showing lucide `Camera` icon (32px, forest-mid) center. On hover: border solid, clay-tinted `bg-clay/5`. After upload: shows the cropped preview (no dashed border), with a small lucide `Pencil` icon (16px, forest on sage-pale circle) in the bottom-right corner as the "change avatar" affordance. Below the avatar: filename `avatar.jpg ¬∑ 87 KB` in 14px muted, with a "Remove" ghost-link in 14px forest-mid.

**County Combobox:** Trigger is a shadcn `Button variant="outline"` with a right-aligned `ChevronsUpDown` icon (lucide, 16px muted). When no county selected, placeholder text "Select county" in muted; when selected, "Fulton County" in forest-deep. Popover opens a `Command` list with a `CommandInput` placeholder "Search counties‚Ä¶" (autofocus on open) and 159 `CommandItem` rows, each `h-9` with the county name left-aligned and a checkmark icon on the selected row. Empty state inside the combobox (user types "zz"): `CommandEmpty` shows "No county matches 'zz'. Try another spelling." in 14px muted.

**Category selector:** shadcn `RadioGroup` rendered as a 2-column grid (`grid-cols-2`) on mobile, 3-column on `md+`, of button-style tiles. Each tile: `h-16`, `border` 1px sage-light ‚Üí clay on selected, `rounded-md`, centered label text (14px Inter, truncating at ellipsis if needed). Selected tile gets `bg-clay/10 border-clay text-forest-deep`. Only one tile can be selected at a time.

**Success toast (after save):** shadcn `Sonner` (`toast.success`) with the copy `Profile saved.` ‚ÄĒ no emoji, no trailing exclamation. Background sage-pale, 1px sage-light border, 6px rounded, forest-deep text, lucide `Check` icon 16px forest-mid inline-left, positioned `bottom-right` on desktop / `top-center` on mobile (auto by Sonner). Auto-dismiss 3s. Toast container mounted once in `app/(app)/layout.tsx`.

## Accessibility Contract

- All form inputs have `<Label htmlFor>` wired via shadcn `Form` primitives (inherits Phase 2 pattern).
- Error messages use `aria-describedby` via shadcn `FormMessage`.
- Skill-row remove buttons use `aria-label="Remove skill {n}"` since the visible label is an `√ó` icon.
- Avatar drop-zone is a keyboard-focusable `<button>` that opens the native file picker; the dashed placeholder has `aria-label="Upload profile photo"`; after upload, the wrapper switches to `aria-label="Change profile photo ‚ÄĒ current photo is avatar.jpg, {size} KB"`.
- Publish Switch has `aria-describedby` pointing to the checklist when disabled (so screen readers announce the unmet requirements).
- County Combobox uses shadcn `Command` default ARIA (listbox + option roles); the trigger is a `<button aria-expanded>`.
- Category RadioGroup uses shadcn default (`role="radiogroup"` + `role="radio"` per tile).
- Focus rings: 2px clay, 2px offset (Phase 2 `--color-ring` unchanged).
- Color contrast:
  - Forest-deep `#1e4420` on sage-pale `#f4f7f0` (body text in editor card) = 10.2:1 (AAA)
  - Forest-mid `#3a7032` on sage-bg `#eef3e8` (muted-foreground) = 5.4:1 (AA pass, AAA fail for small text ‚ÄĒ acceptable at 16px+)
  - Clay `#c4956a` on sage-bg (button bg) with sage-bg `#eef3e8` foreground ‚Üí inverted; actual button is clay bg + sage-bg text = 6.9:1 (AAA)
  - Destructive (shadcn default) on sage-pale card bg = 4.7:1 (AA for body text pass)
- Skip-link is NOT added in Phase 3 (app shell is short; revisit Phase 6 if landing page grows).
- All interactive controls are reachable by keyboard; tab order follows visual order top‚Üíbottom left‚Üíright.
- Touch targets ‚Č•44px on all interactive controls (see Spacing exceptions).
- `prefers-reduced-motion`: the Switch, Combobox open/close, and Sonner toasts respect the OS-level reduced-motion preference (shadcn defaults honor this via Radix ‚ÄĒ no extra work required; executor confirms during UAT).
