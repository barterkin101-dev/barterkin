## Spacing Scale

Declared values (Tailwind default 4-pt scale; admin uses only these):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`gap-1`, `p-1`) | Icon-to-label gap inside badges and buttons |
| sm | 8px (`gap-2`, `p-2`) | Compact element spacing; stats-card inner rhythm |
| md | 16px (`gap-4`, `p-4`) | Default element spacing; table cell padding; card body padding |
| lg | 24px (`gap-6`, `p-6`) | Admin page gutter (`px-6`); section padding |
| xl | 32px (`gap-8`, `py-8`) | Stats-grid vertical rhythm |
| 2xl | 48px (`py-12`) | Between hero header and stats grid on `/admin` home |
| 3xl | 64px (`py-16`) | Reserved — not used in admin phase |

Exceptions:
- **Touch targets:** All tappable controls (ban/unban buttons, member row, tab triggers, search clear) render at ≥44px tall. Use shadcn Button `size="default"` (40px) upgraded to `size="lg"` (44px) or `h-11` for primary admin actions — matches the `Build your profile` pattern in `DirectoryEmptyState.tsx`.
- **Members list row:** Each row has `py-3` (12px) top/bottom padding on a 48px min-height row — an intentional deviation from the pure 8-point scale to keep the table dense enough for a non-technical scanner and still hit the 44px touch target.

Max content width: `max-w-5xl` (matches `(app)` layout) with `px-6` gutter.

## Typography

Exactly 4 sizes, exactly 2 weights (regular 400 + bold 700). Mirrors the directory page pattern (`page.tsx` uses `font-serif text-[32px] font-bold` for H1, `text-base` body).

| Role | Size | Weight | Line Height | Family |
|------|------|--------|-------------|--------|
| Display (page H1) | 32px (`text-[32px]`) | 700 (`font-bold`) | 1.15 (`leading-[1.15]`) | Lora (`font-serif`) |
| Heading (card title, section H2) | 20px (`text-xl`) | 700 (`font-bold`) | 1.2 (`leading-tight`) | Lora (`font-serif`) |
| Body (default text, table cells, dialog body) | 16px (`text-base`) | 400 (`font-normal`) | 1.5 (`leading-normal`) | Inter (`font-sans`) |
| Label (table headers, badges, helper copy) | 14px (`text-sm`) | 400 or 700 | 1.5 | Inter (`font-sans`) |

Rules:
- Headings and display: Lora / forest-deep.
- Body and labels: Inter.
- Never use a 5th size; never use weight 500 or 600 (match Phase 4 pattern — directory uses only 400/700).

## Color

60/30/10 split anchored on the existing Barterkin palette.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `bg-sage-bg` `#eef3e8` | Admin page background; inherits from `bg-sage-bg` on the admin layout wrapper |
| Secondary (30%) | `bg-sage-pale` `#f4f7f0` + `ring-sage-light` `#dfe8d5` | Stats cards, members-list container, dialog surfaces, tabs bar — matches DirectoryCard pattern |
| Accent (10%) | `bg-clay` `#c4956a` / `text-clay` | Reserved only for: primary action buttons (confirm-ban / confirm-unban in AlertDialog), the selected tab indicator in the contact-requests tabs, and the stat value number on the `/admin` home cards |
| Destructive | `var(--destructive)` (shadcn default `oklch(0.577 0.245 27.325)`) | Only on the "Banned" member status badge and the destructive-variant confirm button inside the ban AlertDialog |
| Text primary | `text-forest-deep` `#1e4420` | All headings, member names, stat numbers |
| Text secondary | `text-forest-mid` `#3a7032` | County labels, join dates, helper copy (matches `muted-foreground` override) |

Accent reserved for:
1. Primary CTA buttons inside AlertDialog (`bg-clay hover:bg-clay/90 text-sage-bg h-11`) for unban confirmation
2. Active tab indicator on the contact-requests Status filter tabs
3. Stat value number (the large number on each stats card — e.g. "47 Total Members")

Accent is NOT used for: navigation links, row hover states, table borders, pagination controls, or the search input focus ring. Those use `forest-mid` or the default `ring-sage-light`.

Status badge colors on the members list:
- **Published:** `bg-sage-light text-forest-deep` — calm, "good state"
- **Unpublished:** `bg-muted text-muted-foreground` — neutral gray
- **Banned:** `bg-destructive/10 text-destructive border border-destructive/20` — only place destructive color appears outside the confirm-ban button

| `skeleton` | Loading state for stats cards and members table (3 rows minimum) |
| `pagination` | Members list if >50 rows (planner decides threshold — suggest 50/page) |
| `separator` | Between admin nav and content; inside member detail sections |
| `sonner` (Toaster) | Ban/unban success + error toasts — already mounted in `app/(app)/layout.tsx`; admin layout should mount its own Toaster |
| `tabs` | **NOT currently installed** — planner must run `pnpm dlx shadcn@latest add tabs` as a Wave-0 dependency for the contact-requests filter tabs |

**Table strategy** (Claude's discretion from CONTEXT.md D-04/D-06): Use a plain HTML `<table>` styled with Tailwind (`bg-sage-pale ring-1 ring-sage-light rounded-lg`) rather than installing `components/ui/table.tsx`. Rationale: minimal admin, only 2 tables (members + contacts), no sorting/filtering complexity that justifies the shadcn table abstraction. Rows are `<Link>` components for keyboard navigation.

**Pagination threshold** (Claude's discretion from CONTEXT.md): Pre-launch seeding target is ≥30 profiles (SEED-05). Steady-state at launch will be <100. Use `pagination` component at 50 rows/page — single page for the foreseeable future, but wire it in now so it doesn't break at scale.

**Admin nav structure** (Claude's discretion from CONTEXT.md D-02): Simple **top nav** (not sidebar). Three links: Overview / Members / Contacts. Matches the minimal, non-technical brief and reuses the existing `AppNav` visual pattern (`bg-sage-pale border-b border-sage-light`) with a distinct `Barterkin Admin` brand lockup so she always knows she's in the moderation surface.

## Interaction Contract

- **Real-time search (members):** No submit button; 300ms debounce; clears with an X button inside the input when a query is present; Escape key also clears.
- **Row click:** Full-row `<Link>` to detail page; focus ring on keyboard navigation (forest-mid outline, matches existing `focus-visible` pattern).
- **Ban flow:** Click `Ban this member` → AlertDialog opens → Confirm fires Server Action using `lib/supabase/admin.ts` service-role client → on success, dialog closes, page revalidates, sonner toast `{displayName} has been banned.`, status pill flips to `Banned`, primary button re-labels to `Unban this member`.
- **Unban flow:** Same pattern, inverse.
- **Tabs (contacts):** URL-state (`?status=bounced`) for shareability and back-button restoration; matches directory filter pattern.
- **Loading states:** Skeleton cards for stats home; 3-row skeleton table for members/contacts; dialog button shows pending state (`Banning…`) during Server Action in flight (reuses `useFormStatus`).
- **Error states:** Inline error card with refresh guidance — no global error boundary swallow.

## Accessibility Contract

- All interactive elements are keyboard-reachable (Tab / Shift+Tab) with visible focus rings (`focus-visible:ring-2 focus-visible:ring-clay ring-offset-2 ring-offset-sage-pale`).
- All icons are decorative unless paired with a label; icon-only buttons have `aria-label`.
- Color is never the sole status indicator — status badges always include text labels (`Published` / `Unpublished` / `Banned`).
- AlertDialog uses Radix's built-in focus-trap + Escape-to-close + `aria-describedby` on the body text.
- Tables use semantic `<table>` / `<thead>` / `<th scope="col">` / `<tbody>` / `<tr>` / `<td>` — no divs-as-tables.
- Minimum tap target 44×44px for all primary admin actions (enforced via `h-11` on the ban/unban buttons and `py-3` rows on the members table).
- Contrast: clay `#c4956a` on sage-bg passes WCAG AA for large text (≥18px bold); body text is forest-deep `#1e4420` on sage-bg, which passes AAA.
